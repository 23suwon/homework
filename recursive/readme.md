## 각 방법 분석

### 재귀적 방법

F(n) = F(n-1) + F(n-2) 라는 수학적 정의를 코드로 그대로 옮긴 방식으로 코드가 직관적이고 짧지만 F(5)를 구할 때 F(3)을 2번, F(2)를 3번 계산하는 것처럼 **이미 계산한 값을 반복해서 다시 계산**하는 문제가 있습니다. 이 때문에 N이 커질수록 함수 호출 횟수가 기하급수적으로 늘어나며, 시간복잡도는 **O(2ⁿ)** 입니다. N이 매우 크면 스택 오버플로우가 발생할 수 있습니다.

| 항목 | 내용 |
|------|------|
| 시간복잡도 | O(2ⁿ) |
| 공간복잡도 | O(n) |
| 장점 | 코드가 직관적, 수학적 정의와 일치 |
| 단점 | 중복 계산, 스택 오버플로우 위험 |

### 순환적 방법

이전 두 값(f1, f2)만 변수에 저장하면서 반복문으로 순차적으로 계산하는 방식입니다 중복 계산이 전혀 없고, 변수 3개(f1, f2, fn)만 사용하므로 공간도 거의 낭비가 없습니다. 시간복잡도는 **O(n)**, 공간복잡도는 **O(1)** 로 매우 효율적입니다.

| 항목 | 내용 |
|------|------|
| 시간복잡도 | O(n) |
| 공간복잡도 | O(1) |
| 장점 | 빠르고 안정적, 스택 오버플로우 없음 |
| 단점 | 재귀에 비해 코드가 덜 직관적 |
